Introduction

This is one of the first complete projects I made with Java for a computer science course at my high school. This project served as my first real introduction to more complicated Java development, as well as the use of Swing to create a GUI for the user to interact with.


Overview

The game is simple; Each player enters their name for the leaderboard, then take turn's placing x's or o's on the board until there is a winner or a draw. After the game is over, the leaderboard will be updated and saved to a file. Player's will be able to play again or close the game.

The Starting Point

The first thing I did was the ever-wonderfull process of creating classes. For a game like TicTacToe, I created Game, Board, Space, Player, and DisplayManager. I wanted to focus on the logic of the game itself. So I decided to make the game work on the command line first. Within about a day of work, the game was playable through the command line. It was time to give my project a fresh coat of paint.


Adding a GUI with Swing

Swing is a Java library that is designed for created interactive apps. Swing was a major challenge for me, partly because of my inexperience with it, but also because of Swing's limits and design.

For the sake of simplicity, I made the Game class control the hierarchy of Swing components. The Space class is itself a button that the player clicks on to mark it. In a larger project, I would have kept the DisplayManager class around to manage the complexity of a GUI.
